Loan Module Developer Guide

This guide describes how developers can run a local chain, configure module parameters, and test the full lifecycle of a loan.

Prerequisites

  • Go installed
  • Cosmos SDK application built
  • CLI binary simd available

Step 1 — Initialize Development Chain

make init-dev

This command will:

  • Remove previous chain state
  • Initialize a new chain
  • Create validator, borrower, laz and omnibus accounts
  • Configure governance voting period
  • Start the node

Step 2 — Submit Governance Proposal

Module parameters must be configured through governance. 

make create-proposal

This proposal updates the module parameters including:

  • Settlement denom
  • Minimum loan amount
  • Maximum loan amount
  • Authorized LAZ addresses
  • Authorized Omnibus addresses

Step 3 — Check Proposal Status

./build/simd query gov proposal 1

Step 4 — Create Loan

./build/simd tx loan create-loan \
--laz <LAZ_ADDRESS> \
--principal 1000stake \
--tenor-months 6 \
--metadata-hash QmHash123 \
--from borrower \
--chain-id overloan-1

Status: PENDING

Step 5 — Approve or Reject Loan

Approve Loan

./build/simd tx loan approve-loan <loan_id> --from laz

Status: APPROVED

Reject Loan

./build/simd tx loan reject-loan <loan_id> --from laz

Status: REJECTED

Step 6 — Confirm Disbursement

./build/simd tx loan confirm-disbursement <loan_id> --from omnibus

Status: DISBURSED

Step 7 — Repay Loan

./build/simd tx loan repay-loan <loan_id> --from omnibus

Status: REPAID

Loan Lifecycle Summary

  1. Create loan request
  2. Approval or rejection by LAZ
  3. Disbursement confirmation by Omnibus
  4. Repayment of outstanding balance
